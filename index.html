Picaas API Documentation

Picaas is an online service using Deep-Learning-Based AI technologies to fix photos in compliance with image requirements of Google Shopping Ads.

Check offical web site ( https://picaas.io ) to get more informatoin about Picaas

Quick Start

To get started, you should have client_id and client_secret for your service. If you havenâ€™t had, please contact us to create for you.

You have to use client_id and client_secret to get an application token. and place token at every API request header for authentication.

API detail

Authentication

Create application token for authentication. The token is used in the following APIs and put in BEARER header. Use client_id and client_secret provided by us.

Endpoint

POST https://app.straas.net/api/v1/app/token

Parameters

Name Description Type
client_id Application client id. String
client_secret Application client secret. String

Request

Route

POST https://app.straas.net/api/v1/app/token

Headers

Content-Type: application/json

Body

{
    "client_id": "<INPUT_YOUR_CLIENT_ID_HERE>",
    "client_secret": "<INPUT_YOUR_CLIENT_SECRET_HERE>"
}

Response

Response Fields

Name Description Type
account_id The id of the account which the resource belongs to. String
token JSON Web Token. A app token can be alive in 24 hours. After it is expired or invalid, you have to get a new one. String

Status

201 Created

Body

{
    "token": "<APP_TOKEN>",
    "account_id": "picaas.io-test"
}

Create Image Batch Job

Create image batch jobs. There are three types of image batch job.

  • pre_validation: Judge the image is legal or illegal. Use it when image wan't legalized by Picaas.
  • post_validation: Judge the image is legal or illegal. Use it when image was legalized by Picaas at least once.
  • legalization: Picaas will make images legal by editing image automatically.

Endpoint

POST https://app.picaas.io/v1/image_legalization_pipelines

Parameters

Name Description Type
model_type The model type includes { legalization | pre_validation | post_validation }. String
input_csv Base64 encode CSV flie. String

Format of input_csv

Name Description Type
custom_id Custom id that you can manage. String
url The url of image you want to submit. String

You don't have to add header of CSV. Just fill in the custiom_id and url in order. Check example below. 

ID001, https://imgur.com/mZ7AjDP.jpg
ID002, https://imgur.com/pfTZazm.jpg

Remember to convert CSV to base64 before submit. You can test with CSV to base64 converter. The example below is encoded base 64 string from CSV. 

SUQwMDEsIGh0dHBzOi8vaW1ndXIuY29tL21aN0FqRFAuanBnCklEMDAyLCBodHRwczovL2ltZ3VyLmNvbS9wZlRaYXptLmpwZw==

Request

Route

POST https://app.picaas.io/v1/image_legalization_pipelines

Headers

Authorization: Bearer <APP_TOKEN>
Content-Type: application/json

Body

{
    "model_type":"legalization",
    "input_csv":"MSxodHRwczovL2ltZ3VyLmNvbS9tWjdBakRQLmpwZwoyLGh0dHBzOi8vaW1ndXIuY29tL3BmVFphem0uanBn"
}

Response

Response Fields

Name Description Type
task[status_url] use the url to query image batch job status. String
task[id] The id of the batch job. String

Status

202 Accepted

Body

{
    "task": {
        "status_url": "/image_legalization_pipelines/5566/status",
        "id": 5566
    }
}

Get Image Batch Job Status

Check the current status of image batch job.

Endpoint

GET https://app.picaas.io/v1/image_legalization_pipelines/:id/status

Parameters

Name Description Type
id The id of the batch job. Integer

Request

Route

GET https://app.picaas.io/v1/image_legalization_pipelines/5566/status

Headers

Authorization: Bearer <APP_TOKEN>
Content-Type: application/json

Response

Response Fields

Name Description Type
status The status includes { completed | initialized | processing }. String
progress The progress of image batch job. Integer
total_image_count The number of total images which are computed. Integer

Status

200 OK

Body

{
    "status": "completed",
    "progress": 100,
    "total_image_count": 2
}

Get Image Batch Job

After the job is completed, you can get the CSV download url (i.e. output_signed_url) by this API. If you get the resource before the job is done, it returns 404 http status code.

Endpoint

GET https://app.picaas.io/v1/image_legalization_pipelines/:id

Parameters

Name Description Type
id The id of the batch job. Integer

Request

Route

GET https://app.picaas.io/v1/image_legalization_pipelines/5566

Headers

Authorization: Bearer <APP_TOKEN>
Content-Type: application/json

Response

Response Fields

Name Description Type
id The id of the batch job. Integer
account_id The id of the account which the resource belongs to. String
model_type The model type includes { legalization | pre_validation | post_validation }. String
status The status includes { completed | initialized | processing }. String
output_signed_url The url of legalization or validation result. It is CSV format. String
completed_at The batch job completed time. String
total_image_count The number of total images which are computed. Integer
succeeded_image_count The number of images which are computed successfully. Integer

Status

200 OK

Body

{
    "id": 5566,
    "account_id": "picaas.io-test",
    "model_type": "legalization",
    "status": "completed",
    "output_signed_url": "https://storage.googleapis.com/picaas-lenna-production-as/data/picaas.io-test......",
    "completed_at": "2019-02-12T14:00:00Z",
    "total_image_count": 2,
    "succeeded_image_count": 2
}

Format of output_signed_url Response

Name Description Type
id The id of the image serial number of Picaas. Integer
custom_id Custom id that your can manage. String
status The status includes { succeeded | failed }. String
error_code Check error code table to know what kind of error happend. String
model_version validation model or legalization model verision. String
output_url The url of legalized image. String
prediction The prediction type includes { acceptable | correctable | uncorrectable }.
  • acceptable: the image is legal.
  • correctable: The image have to be legalized.
  • uncorrectable: The image have to be legalized.
